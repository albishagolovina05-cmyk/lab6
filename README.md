## Вариант задания (16. C#)

Грамматика арифметических выражений языка C#:

| Продукция            | Описание                          |
|----------------------|-----------------------------------|
| `E → T A`            | Выражение → терм + хвост           |
| `A → ε  \| + T A \| - T A` | Хвост – цепочка сложений/вычитаний |
| `T → F B`            | Терм → фактор + хвост умножения    |
| `B → ε  \| * F B \| / F B \| % F B` | Умножение, деление, остаток |
| `F → num \| id \| ( E )` | Фактор – число, идентификатор или подвыражение |
| `id → letter {letter \| digit \| _}` | Идентификатор |
| `num → digit {digit}`               | Целое число без знака |

**Аксиома:** `E`  
**Терминалы:**  
`{ a-z, A-Z, 0-9, _, +, -, *, /, %, (, ) }`  
**Нетерминалы:**  
`{ E, A, T, B, F, id, num }`

---

## Классификация по Хомскому

Грамматика является **контекстно-свободной** (КС-грамматика типа 2).  
Все правила имеют вид `A → α`, где `A ∈ VN`, `α ∈ V*`.  
Применяется **метод рекурсивного спуска** (нисходящий разбор, без возвратов).

---

## Диаграмма лексера 



Лексический анализатор распознаёт:
- идентификаторы (буква → буква/цифра/`_`);
- целые числа (цифра → цифра);
- операторы `+`, `-`, `*`, `/`, `%`, `(`, `)`;
- пропускает пробелы и переводы строк;
- сообщает о недопустимых символах.

---

## Схема рекурсивного спуска для парсера



Функции парсера:
- `E()`  → вызывает `T()` и `A()`
- `A()`  → если текущий токен `+` или `-`, прочитать его, затем `T()` и снова `A()`
- `T()`  → вызывает `F()` и `B()`
- `B()`  → если токен `*`, `/` или `%`, прочитать его, затем `F()` и снова `B()`
- `F()`  → если `id` или `num` – прочитать; если `(` – прочитать, вызвать `E()`, проверить `)`

При ошибке выдаётся сообщение с ожидаемым символом и позицией.

## Внутренняя форма представления программы (тетрады)

Тетрада – это четверка: `(операция, аргумент1, аргумент2, результат)`.

### Пример №1 

**Выражение:** `(x + y) * z`  
**Тетрады:**

| №   | Операция | arg1 | arg2 | результат |
|-----|----------|------|------|-----------|
| 1   | +        | x    | y    | T1        |
| 2   | *        | T1   | z    | T2        |

### Пример №2 

**Выражение:** `a - b / c + d % e`  
**Тетрады:**

| №   | Операция | arg1 | arg2 | результат |
|-----|----------|------|------|-----------|
| 1   | /        | b    | c    | T1        |
| 2   | %        | d    | e    | T2        |
| 3   | -        | a    | T1   | T3        |
| 4   | +        | T3   | T2   | T4        |

### Пример №3 

**Выражение:** `10 * (3 + 2) % 4`  
**Тетрады:**

| №   | Операция | arg1 | arg2 | результат |
|-----|----------|------|------|-----------|
| 1   | +        | 3    | 2    | T1        |
| 2   | *        | 10   | T1   | T2        |
| 3   | %        | T2   | 4    | T3        |

## ПОЛИЗ 

ПОЛИЗ – постфиксная запись, где оператор следует за операндами.  
Вычисление производится с использованием стека.

### Пример №1

**Выражение:** `(x + y) * z`  
**ПОЛИЗ:** `x y + z *`  
**Вычисление:**  
1. поместить `x`, `y` → выполнить `+` → стек: `[x+y]`  
2. поместить `z` → выполнить `*` → результат `(x+y)*z`

### Пример №2

**Выражение:** `a - b / c + d % e`  
**ПОЛИЗ:** `a b c / - d e % +`  
**Вычисление:** (значения: a=10, b=6, c=2, d=8, e=3)  
- `b c /` → 6/2=3  
- `a 3 -` → 10-3=7  
- `d e %` → 8%3=2  
- `7 2 +` → 9  

**Результат:** 9

### Пример №3

**Выражение:** `10 * (3 + 2) % 4`  
**ПОЛИЗ:** `10 3 2 + * 4 %`  
**Вычисление:**  
1. 3 2 + → 5  
2. 10 5 * → 50  
3. 50 4 % → 2  

**Результат:** 2
